# 任务: 排查并修复上游 Mock 401 鉴权失败

## 任务描述
智能入湖平台后端调用上游 mock_service 接口出现 401，需要确认鉴权链路、token 刷新策略与配置一致性，并完成修复与回归验证。

## 完成内容

### 1. 问题复现与定位
- 现象：智能入湖平台调用 mock_service 受保护接口偶发/持续返回 401。
- 定位：
  - `TokenProvider` 仅在首次取到 token 后永久缓存，不会根据 `expires_in` 刷新，token 过期后继续使用将导致 401。
  - 客户端请求遇到 401 时不会主动清理 token 并重试，容易出现“过期后一直 401”。
- 关联代码：`TokenProvider.get_token()` 与 `BaseClient.request()`。

### 2. 修复实现
- 文件: `app/core/dependencies.py`
  - `TokenProvider` 增加 `expires_in` 解析与 `_expires_at`，在过期前自动刷新 token。
  - 注入到各平台客户端的 `token_provider` 改为绑定方法 `get_token_provider().get_token`，便于在 401 时清缓存。
- 文件: `app/core/clients/base_client.py`
  - 当收到 401 且 `token_provider` 为可清缓存的绑定方法时：清 token 并重试一次。
- 文件: `tests/unit/test_token_provider.py`
  - 新增单元测试覆盖“过期刷新/未过期不刷新”。

### 3. 回归验证
- 执行: `PYTHONPATH=. pytest tests/unit/test_token_provider.py`
- 结果: `2 passed`
- 执行: `PYTHONPATH=. pytest tests/unit/test_dependencies.py tests/unit/test_chat_agent.py tests/integration/test_config_service.py`
- 结果: `26 passed`
- 手动验证：构造无效 token 后请求 mock 受保护接口，确认 401 后自动刷新并成功返回。
