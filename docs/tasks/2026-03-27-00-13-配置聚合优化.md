# 配置聚合优化

**任务时间**: 2026-03-27 00:13

## 任务目标

优化配置管理模块，将平铺的配置属性改为聚合的配置节，提升代码的可维护性和优雅性。

## 问题分析

原有设计存在以下问题：

1. **配置平铺**: 所有配置属性都直接挂在Settings类上，如`settings.file_upload_enabled`、`settings.file_upload_max_size`等
2. **代码冗余**: 每个property都需要重复写`config = self._load_config()`和`config.get("section", {}).get("key")`
3. **不够优雅**: 缺乏层次结构，配置项之间的关系不清晰

## 解决方案

### 设计思路

引入**配置节（ConfigSection）**概念，将相关配置聚合到一起：

```
settings
├── app (AppConfig)
│   ├── name
│   ├── version
│   └── debug
├── server (ServerConfig)
│   ├── host
│   └── port
├── file_upload (FileUploadConfig)
│   ├── enabled
│   ├── max_size
│   ├── allowed_extensions
│   ├── temp_dir
│   └── cleanup_hours
└── ...
```

### 实现细节

#### 1. 配置节基类

```python
class ConfigSection:
    """配置节基类"""
    
    def __init__(self, config: Dict[str, Any], section_name: str):
        self._config = config
        self._section_name = section_name
    
    def _get(self, key: str, default: Any = None) -> Any:
        return self._config.get(self._section_name, {}).get(key, default)
```

**优点**:
- 封装了`config.get()`的重复逻辑
- 子类只需调用`self._get(key)`即可获取配置
- 统一的默认值处理

#### 2. 具体配置节类

```python
class FileUploadConfig(ConfigSection):
    """文件上传配置"""
    
    @property
    def enabled(self) -> bool:
        return self._get("enabled", False)
    
    @property
    def max_size(self) -> int:
        return self._get("max_size")
    
    @property
    def cleanup_hours(self) -> int:
        return self._get("cleanup_hours")
```

**优点**:
- 代码简洁，每个property只需一行
- 配置项聚合，语义清晰
- 易于扩展和维护

#### 3. Settings类改造

```python
class Settings:
    def __init__(self):
        # ...
        self._app: Optional[AppConfig] = None
        self._file_upload: Optional[FileUploadConfig] = None
        # ...
    
    @property
    def app(self) -> AppConfig:
        if self._app is None:
            self._app = AppConfig(self._load_config(), "app")
        return self._app
    
    @property
    def file_upload(self) -> FileUploadConfig:
        if self._file_upload is None:
            self._file_upload = FileUploadConfig(self._load_config(), "file_upload")
        return self._file_upload
    
    @property
    def file_upload_cleanup_hours(self) -> int:
        return self.file_upload.cleanup_hours
```

**特点**:
- 延迟初始化配置节对象
- 提供聚合访问方式：`settings.file_upload.cleanup_hours`
- 保留兼容性属性：`settings.file_upload_cleanup_hours`

## 使用示例

### 推荐方式：聚合访问

```python
from app.settings import settings

print(settings.app.name)
print(settings.app.version)
print(settings.app.debug)

print(settings.server.host)
print(settings.server.port)

print(settings.file_upload.enabled)
print(settings.file_upload.max_size)
print(settings.file_upload.cleanup_hours)
```

### 兼容方式：平铺访问

```python
from app.settings import settings

print(settings.app_name)
print(settings.debug)
print(settings.file_upload_cleanup_hours)
```

## 优势对比

### 重构前

```python
@property
def file_upload_cleanup_hours(self) -> int:
    config = self._load_config()
    return config.get("file_upload", {}).get("cleanup_hours")
```

**问题**:
- 每个property都要重复`config = self._load_config()`
- 每个property都要重复`config.get("section", {}).get("key")`
- 配置项分散，不易管理

### 重构后

```python
class FileUploadConfig(ConfigSection):
    @property
    def cleanup_hours(self) -> int:
        return self._get("cleanup_hours")
```

**优势**:
- 代码简洁，一行搞定
- 配置聚合，语义清晰
- 易于扩展和测试

## 代码质量提升

- **模块化**: 每个配置节独立管理
