# 实现通用批量处理能力

**任务时间**: 2026-03-24 21:12

## 任务目标

基于LangGraph子图机制，实现通用的批量处理能力，用于处理需要对多个项目执行相同操作的场景（如批量字段映射）。核心设计理念是通过独立子会话隔离上下文，避免批量处理时的上下文累积问题。

## 需求分析

### 问题背景

用户需要批量处理多个单号的字段映射任务。如果在同一个LangGraph会话中循环处理，会导致：
1. **上下文累积**：每次循环的消息都会追加到历史中
2. **性能下降**：处理到后面的单号时，上下文已经非常庞大
3. **Token消耗巨大**：可能超出模型上下文限制

### 设计目标

1. **独立上下文**：每个批次项在独立子会话中处理
2. **通用能力**：不局限于field-mapping，任何批量场景都可使用
3. **代码层实现**：核心逻辑在代码中，Skill只需调用工具
4. **零上下文累积**：子会话独立，不影响主会话

## 设计方案

### 架构设计

```
┌─────────────────────────────────────────────┐
│         主Agent（DeepAgents）                │
│  - 解析Excel获取任务列表                     │
│  - 调用batch_process工具                    │
└──────────────────┬──────────────────────────┘
                   │
                   ↓
┌─────────────────────────────────────────────┐
│      batch_process Tool                     │
│  - 接收任务列表和处理指令                    │
│  - 为每个任务创建独立子会话                  │
│  - 循环调用子Agent处理                       │
│  - 汇总结果返回                              │
└──────────────────┬──────────────────────────┘
                   │
        ┌──────────┴──────────┐
        ↓                     ↓
┌──────────────┐      ┌──────────────┐
│ 子Agent会话1  │      │ 子Agent会话2  │
│ (独立上下文)  │      │ (独立上下文)  │
│ 处理ORDER001 │      │ 处理ORDER002 │
└──────────────┘      └──────────────┘
```

### 核心组件

1. **batch_process工具**：LangChain Tool，提供批量处理能力
2. **excel_parser工具**：通用Excel解析工具，将Excel转为JSON
3. **独立子会话**：每个批次项使用独立的thread_id

## 实现内容

### 1. 批量处理工具

**文件**: `app/agents/tools/batch_tool.py`

核心功能：
- 接收JSON格式的任务列表
- 使用指令模板为每个任务生成处理指令
- 为每个任务创建独立子会话（独立thread_id）
- 调用DeepAgent处理每个任务
- 汇总所有结果并返回

关键特性：
- 支持自动分批（默认每批5个）
- 容错处理（单个失败不影响其他）
- 详细的结果统计

### 2. Excel解析工具

## 使用示例

### Agent使用批量处理工具

```python
# 1. 解析Excel
result = execute_command("python3 skills/common-tools/excel_parser.py uploaded_file.xlsx")
data = json.loads(result)

# 2. 调用batch_process工具
batch_process(
    items=json.dumps(data['data']),
    instruction_template='处理单号{单号}的字段映射，源库为{源库}，源表为{源表}',
    batch_size=5
)
```

### 工具返回结果

```json
}
```

## 技术亮点

1. **独立子会话隔离**：每个批次项使用独立的thread_id，完全隔离上下文
2. **通用性强**：不局限于特定业务，任何批量场景都可使用
3. **代码层实现**：核心逻辑在代码中，Skill只需调用工具，保持轻量
4. **自动分批**：超过batch_size自动分批，避免一次处理过多
5. **容错机制**：单个项目失败不影响其他项目处理
6. **详细统计**：返回每个项目的处理状态和汇总统计
