# 前端中断处理实现

**任务时间**: 2026-03-26 01:00

## 问题描述

后端已经实现了中断检测和 Resume API，但前端没有实现 interrupt 事件的处理，导致用户无法看到确认对话框。

## 实现内容

### 1. 状态管理

**文件**: `frontend/src/components/Chat.jsx`

添加了中断相关的状态：

```javascript
// 中断状态管理
const [pendingInterrupt, setPendingInterrupt] = useState(null); // { interrupt_info, thread_id, tool_call }
const lastToolCallRef = useRef(null); // 保存最后的工具调用
```

### 2. 事件处理

#### 2.1 保存工具调用信息

在 `tool_call` 事件处理中保存最后的工具调用：

```javascript
} else if (event.type === 'tool_call') {
```

#### 2.2 处理 interrupt 事件

添加了 `interrupt` 事件的处理逻辑：

```javascript
} else if (event.type === 'interrupt') {
  // 中断事件 - 需要用户确认
  const { interrupt_info, thread_id } = event;
  
  // 保存中断信息和最后的工具调用
  setPendingInterrupt({
    interrupt_info,
    thread_id,
    tool_call: lastToolCallRef.current // 保存最后的工具调用信息
  });
  
  // 停止思考动画
  if (thinkingUpdateTimerRef.current) {
    clearTimeout(thinkingUpdateTimerRef.current);
    thinkingUpdateTimerRef.current = null;
  }
  updateThinkingMessage(false);
}
```

### 3. Resume API 调用

实现了 `handleResumeExecution` 函数：

```javascript
const handleResumeExecution = async (decision) => {
  if (!pendingInterrupt) return;
  
  try {
    const response = await fetch('http://localhost:8000/chat/resume', {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        thread_id: pendingInterrupt.thread_id,
        decision: decision
      })
    });
    
    const result = await response.json();
    
    if (result.success) {
      // 添加结果消息
      const resultMessage = {
        role: 'assistant',
        content: result.message,
        timestamp: new Date().toISOString(),
      };
      setMessages(prev => [...prev, resultMessage]);
    } else {
      // 显示错误
      const errorMessage = {
        type: 'error',
        content: result.message || '操作失败',
        timestamp: new Date().toISOString(),
      };
      setMessages(prev => [...prev, errorMessage]);
    }
  } catch (error) {
    console.error('Resume execution failed:', error);
    const errorMessage = {
      type: 'error',
      content: `恢复执行失败: ${error.message}`,
      timestamp: new Date().toISOString(),
    };
    setMessages(prev => [...prev, errorMessage]);
  } finally {
    // 清除中断状态
    setPendingInterrupt(null);
  }
};
```

## UI 设计

### 确认对话框特点

1. **全屏遮罩**: 半透明黑色背景，防止用户误操作
2. **居中显示**: 对话框居中显示，最大宽度 600px
3. **蓝色边框**: 2px 蓝色边框，突出显示
4. **工具信息展示**:
   - 显示工具名称（如 `platform_service`）
   - 显示工具参数（JSON 格式，可滚动）
5. **操作按钮**:
   - ❌ 拒绝按钮（灰色，subtle 样式）
   - ✅ 确认按钮（蓝色，filled 样式）

## 完整流程

```
1. 用户发送消息："帮我在 dws 数据库执行 DDL"
   ↓
2. 后端智能体调用 platform_service 工具
   ↓
3. 后端检测到需要确认（REQUIRES_CONFIRMATION: true）
   ↓
4. 后端发送 interrupt 事件（SSE）
   ↓
5. 前端收到 interrupt 事件
   ↓
6. 前端保存中断信息到 pendingInterrupt 状态
   ↓
7. 前端显示确认对话框（模态窗口）
   ↓
8. 用户查看工具调用信息
   ↓
9. 用户点击"确认"或"拒绝"
   ↓
10. 前端调用 POST /chat/resume API
    - 传递 thread_id 和 decision
   ↓
11. 后端恢复执行
    - approve: 继续执行工具调用
    - reject: 终止执行
   ↓
12. 后端返回结果
   ↓
13. 前端显示结果消息
   ↓
14. 前端清除 pendingInterrupt 状态
```
