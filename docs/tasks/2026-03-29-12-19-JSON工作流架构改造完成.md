# JSON工作流架构改造完成

**任务时间**: 2026-03-29 12:19

## 改造目标

将工作流从硬编码的Python代码改造为**JSON配置 + Python脚本**的形式,实现:
- 工作流配置化,易于维护和扩展
- 脚本模块化,每个节点独立
- 作为Skill的一部分,符合Skill理念

## 新架构设计

### 目录结构

```
skills/scenario-skill/field-mapping/
├── SKILL.md              # Skill文档
├── workflow.json         # 工作流配置(节点、边、参数)
└── scripts/              # Python脚本
    ├── get_source_schema.py
    ├── generate_csv.py
    ├── run_mapping.py
    ├── read_ddl.py
    └── validate.py
```

### 工作流配置 (workflow.json)

```json
{
  "name": "field-mapping",
  "version": "1.0.0",
  "description": "字段映射工作流 - 将源表字段映射到目标湖表",
  "parameters": {
    "order_id": {"type": "string", "required": true},
    "source_db": {"type": "string", "required": true},
    ...
  },
  "nodes": [
    {
      "id": "get_source_schema",
      "name": "获取源表结构",
      "script": "scripts/get_source_schema.py",
      "progress": {
        "started": "正在获取源表结构",
        "completed": "源表结构获取成功",
        "failed": "源表结构获取失败"
      },
      "inputs": ["source_db", "source_table"],
      "outputs": ["source_schema"]
    },
    ...
  ],
  "edges": [
    {"from": "get_source_schema", "to": "generate_csv"},
    ...
  ],
  "entry_point": "get_source_schema"
}
```

### Python脚本规范

每个脚本:
- 从stdin读取JSON输入
- 输出JSON结果到stdout
- 独立可测试

```python
#!/usr/bin/env python3
import json
import sys

def main():
    inputs = json.loads(sys.stdin.read())
    
    # 处理逻辑
    result = do_something(inputs)
    
    # 输出结果
    print(json.dumps({
        "success": True,
        "outputs": {
            "key": "value"
        }
    }, ensure_ascii=False))

if __name__ == "__main__":
    main()
```

## 架构对比

### 改造前: 硬编码Python

```python
# app/workflows/field_mapping_workflow.py (200+ 行)

def node_get_source_schema(state):
    # 硬编码逻辑
    ...

def node_generate_csv(state):
    # 硬编码逻辑
    ...

@register_workflow("field-mapping")
def build_field_mapping_workflow():
    workflow = StateGraph(...)
    workflow.add_node("get_source_schema", node_get_source_schema)
    ...
    return workflow.compile()
```

**问题**:
- 工作流逻辑和执行引擎耦合
- 修改流程需要改代码
- 不符合Skill的配置化理念
- 难以可视化和管理

### 改造后: JSON配置 + 脚本

```
skills/scenario-skill/field-mapping/
├── workflow.json         # 配置(30行)
└── scripts/              # 脚本(每个20-60行)
    ├── get_source_schema.py
    ├── generate_csv.py
    ├── run_mapping.py
    ├── read_ddl.py
    └── validate.py
```

**优势**:
- ✅ 配置和代码分离
- ✅ 脚本模块化,独立可测试
- ✅ 易于维护和扩展
- ✅ 符合Skill理念
- ✅ 可视化友好
- ✅ 支持热更新(修改JSON无需重启)

## 核心优势

### 1. 配置化

**工作流定义**:
- 节点、边、参数都在JSON中定义
- 修改流程只需编辑JSON
- 无需重新编译

**进度消息**:
```json
"progress": {
  "started": "正在获取源表结构",
  "completed": "源表结构获取成功",
  "failed": "源表结构获取失败"
}
```

### 2. 模块化

**每个脚本独立**:
- 单一职责
- 独立测试
- 易于调试
- 可复用

**标准接口**:
```
输入: JSON (stdin)
输出: JSON (stdout)
```

### 3. 可扩展

**添加新工作流**:
1. 创建目录: `skills/scenario-skill/my-workflow/`
2. 编写配置: `workflow.json`
3. 编写脚本: `scripts/*.py`
4. 自动加载,无需修改代码

**修改现有工作流**:
1. 编辑`workflow.json`添加/删除节点
2. 添加/修改对应的脚本
3. 重启服务即可

### 4. 易于维护

**清晰的结构**:
```
workflow.json  → 一目了然的流程定义
scripts/       → 模块化的实现
SKILL.md       → 完整的文档
```

**版本管理**:
```json
{
  "name": "field-mapping",
  "version": "1.0.0",
  ...
}
```

## 使用示例

### 创建新工作流

```bash
# 1. 创建目录
mkdir -p skills/scenario-skill/my-workflow/scripts

# 2. 创建workflow.json
cat > skills/scenario-skill/my-workflow/workflow.json <<EOF
{
  "name": "my-workflow",
  "version": "1.0.0",
  "description": "我的工作流",
  "parameters": {...},
  "nodes": [...],
  "edges": [...],
  "entry_point": "step1"
}
EOF

# 3. 创建脚本
cat > skills/scenario-skill/my-workflow/scripts/step1.py <<EOF
#!/usr/bin/env python3
import json
import sys

def main():
    inputs = json.loads(sys.stdin.read())
    # 处理逻辑
    print(json.dumps({"success": True, "outputs": {...}}))

if __name__ == "__main__":
    main()
EOF

# 4. 重启服务,自动加载
```

### 修改现有工作流

```bash
# 1. 编辑配置
vim skills/scenario-skill/field-mapping/workflow.json

# 2. 修改/添加脚本
vim skills/scenario-skill/field-mapping/scripts/new_step.py

# 3. 重启服务
```
