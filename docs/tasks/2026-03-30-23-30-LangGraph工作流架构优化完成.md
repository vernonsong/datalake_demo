# LangGraph工作流架构优化完成

**任务时间**: 2026-03-30 23:30

## 优化目标

基于用户反馈,对工作流架构进行优化:

1. **使用LangGraph** - 支持条件边、人工审批等高级特性
2. **脚本纯逻辑化** - 不允许耦合IT系统(如请求上游)
3. **工具调用模式** - 像platform_service那样,通过工具+hook调用外部系统

## 新架构设计

### 核心理念

```
workflow.json (配置)
    ↓
LangGraph (执行引擎)
    ↓
├─ 纯逻辑脚本 (数据处理、验证)
└─ 工具调用 (IT系统交互)
```

### 节点类型

1. **script节点** - 纯逻辑脚本
   - 数据转换
   - 格式验证
   - 业务规则检查
   - **不允许**请求外部系统

2. **tool节点** - 工具调用
   - 调用platform_service
   - 传入平台、方法、端点
   - 传入hook代码(DOC_GUARD)

3. **human节点** - 人工审批
   - 等待人工输入
   - 支持审批/拒绝

### 边类型

1. **normal边** - 普通边,直接连接
2. **conditional边** - 条件边,根据状态字段判断

### 2. LangGraph工作流构建器 ✅

文件: [langgraph_workflow_builder.py](file:///Users/vernonsong/Documents/项目/datalake_demo/app/workflows/langgraph_workflow_builder.py)

**核心功能**:

#### 动态构建LangGraph
```python
class LangGraphWorkflowBuilder:
    def build(self):
        state_class = self._create_state_class()
        graph = StateGraph(state_class)
        
        # 添加节点
        for node in self.config["nodes"]:
            node_func = self._create_node_function(node)
            graph.add_node(node["id"], node_func)
        
        # 添加边
        for edge in self.config["edges"]:
            if edge["type"] == "normal":
                graph.add_edge(from_node, to_node)
            elif edge["type"] == "conditional":
                condition_func = self._create_condition_function(edge["condition"])
                graph.add_conditional_edges(from_node, condition_func, {...})
        
        return graph.compile()
```

#### 支持三种节点类型
```python
def _create_node_function(self, node: dict):
    if node_type == "script":
        return self._create_script_node(...)
    elif node_type == "tool":
        return self._create_tool_node(...)
    elif node_type == "human":
        return self._create_human_node(...)
```

#### 工具调用支持
```python
def _create_tool_node(self, node_id, node, workflow_name):
    tool = self.tools[tool_name]
    
    def tool_node(state):
        # 参数替换: {source_db} → state["source_db"]
        tool_params = {}
        for key, value in tool_config["params"].items():
            if value.startswith("{") and value.endswith("}"):
                param_name = value[1:-1]
                tool_params[key] = state.get(param_name)
            else:
                tool_params[key] = value
        
        result = tool.invoke(tool_params)
        return outputs
    
    return tool_node
```

#### 条件函数支持
```python
def _create_condition_function(self, condition):
    field = condition["field"]
    operator = condition["operator"]
    value = condition["value"]
    
    def condition_func(state):
        state_value = state.get(field)
        
        if operator == "==":
            return state_value == value
        elif operator == "!=":
            return state_value != value
        # 支持 >, <, >=, <=, in, not in
    
    return condition_func
```

### 4. 工作流执行流程 ✅

```
1. get_source_schema (tool)
   ↓ 调用 platform_service
   ↓ 获取源表结构
   
2. generate_csv (script)
   ↓ 纯逻辑:生成CSV
   
3. run_mapping (script)
   ↓ 纯逻辑:运行映射脚本
   
4. read_ddl (script)
   ↓ 纯逻辑:读取DDL文件
   
5. check_ddl_valid (script)
   ↓ 纯逻辑:检查DDL有效性
   ↓ 输出: is_valid
   
6. 条件分支:
   ├─ is_valid == false → human_review (人工审批)
   │                       ↓ 输出: approved
   │                       ├─ approved == true → validate
   │                       └─ approved == false → END
   │
   └─ is_valid == true → validate (验证结果)
                         ↓
                         END
```

## 架构对比

### 改造前: 简单JSON引擎

```python
# 顺序执行,无条件分支
for node in nodes:
    if node["type"] == "script":
        run_script(...)
    # 不支持tool节点
    # 不支持条件边
    # 不支持人工审批
```

**问题**:
- 不支持条件分支
- 不支持人工审批
- 脚本耦合IT系统
- 无法利用LangGraph特性

### 改造后: LangGraph引擎

```python
# LangGraph动态构建
graph = StateGraph(state_class)

# 支持三种节点
graph.add_node("tool_node", tool_func)
graph.add_node("script_node", script_func)
graph.add_node("human_node", human_func)

# 支持条件边
graph.add_conditional_edges(
    "check_node",
    condition_func,
    {True: "next_node", False: END}
)

workflow = graph.compile()
```

**优势**:
- ✅ 支持条件分支
- ✅ 支持人工审批
- ✅ 脚本纯逻辑化
- ✅ 工具调用解耦
- ✅ 利用LangGraph全部特性
