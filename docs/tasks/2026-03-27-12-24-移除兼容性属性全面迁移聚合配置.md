# 移除兼容性属性，全面迁移聚合配置

**任务时间**: 2026-03-27 12:24

## 任务目标

移除Settings类中的所有兼容性属性（如`app_name`、`debug`等），全面迁移到聚合配置方式，使代码更简洁优雅。

## 背景

在之前的配置聚合优化中，我们引入了配置节（ConfigSection）概念，但为了向后兼容，保留了大量平铺的兼容性属性：

```python
@property
def app_name(self) -> str:
    return self.app.name

@property
def debug(self) -> bool:
    return self.app.debug

@property
def file_upload_cleanup_hours(self) -> int:
    return self.file_upload.cleanup_hours
```

这些属性虽然提供了兼容性，但也带来了代码冗余，每个配置都需要写一个委托property。

## 实施方案

### 方案选择

**方案1（之前）**: 保留兼容性属性
- 优点：不需要修改现有代码
- 缺点：Settings类中有大量重复的property定义

**方案2（本次）**: 移除兼容性属性，全面迁移
- 优点：代码更简洁，强制使用更优雅的聚合方式
- 缺点：需要修改所有使用这些属性的地方

选择方案2，理由：
1. 项目还在早期阶段，修改成本可控
2. 聚合方式更优雅，语义更清晰
3. 避免长期维护两套访问方式

### 迁移步骤

#### 1. 移除Settings中的兼容性属性

从Settings类中移除以下属性：
- `app_name` → 使用 `app.name`
- `app_version` → 使用 `app.version`
- `debug` → 使用 `app.debug`
- `host` → 使用 `server.host`
- `port` → 使用 `server.port`
- `mock_service_url` → 使用 `mock_service.url`
- `mock_api_secret` → 使用 `mock_service.api_secret`
- `jwt_secret` → 使用 `jwt.secret`
- `ali_api_key` → 使用 `ali_cloud.api_key`
- `file_upload_enabled` → 使用 `file_upload.enabled`
- `file_upload_max_size` → 使用 `file_upload.max_size`
- `file_upload_allowed_extensions` → 使用 `file_upload.allowed_extensions`
- `file_upload_temp_dir` → 使用 `file_upload.temp_dir`
- `file_upload_cleanup_hours` → 使用 `file_upload.cleanup_hours`

#### 2. 修改所有使用这些属性的代码

需要修改的文件：
1. `app/main.py`

**问题**:
- 14个冗余的委托property
- 每个都是简单的`return self.xxx.yyy`
- 代码重复，维护成本高

### 重构后（Settings类）

```python
class Settings:
    @property
    def file_upload(self) -> FileUploadConfig:
        if self._file_upload is None:
            self._file_upload = FileUploadConfig(self._load_config(), "file_upload")
        return self._file_upload
```

**优势**:
- 移除了14个冗余property
- Settings类更简洁，只保留配置节的property
- 强制使用聚合方式，代码更优雅
