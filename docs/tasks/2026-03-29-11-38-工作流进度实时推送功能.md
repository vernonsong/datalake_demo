# 工作流进度实时推送功能

**任务时间**: 2026-03-29 11:38  
**完成时间**: 2026-03-29 11:40

## 需求背景

用户希望在前端能够实时看到工作流执行到哪一步,而不是等待整个工作流执行完成才看到结果。

## 解决方案

通过在工作流执行过程中发送进度事件,利用SSE流式传输将进度实时推送到前端。

### 架构设计

```
┌─────────────────────────────────────────────────────────┐
│  工作流节点执行                                          │
│  - emit_progress("workflow", "node", "status", data)    │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│  进度回调函数                                            │
│  - print("[WORKFLOW_PROGRESS] {json}")                  │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│  chat_agent 捕获                                         │
│  - 解析 [WORKFLOW_PROGRESS] 标记                        │
│  - yield {type: "workflow_progress", data: {...}}       │
└────────────────────┬────────────────────────────────────┘
                     ↓
┌─────────────────────────────────────────────────────────┐
│  SSE 推送到前端                                          │
│  - 前端实时显示进度                                      │
└─────────────────────────────────────────────────────────┘
```

```python
workflow_progress_matches = re.findall(r'\[WORKFLOW_PROGRESS\]\s*(\{.*?\})', content_str, re.DOTALL)
for progress_json in workflow_progress_matches:
    try:
        progress_data = json.loads(progress_json)
        yield {
            'type': 'workflow_progress',
            'data': progress_data
        }
    except json.JSONDecodeError:
        pass
```

## 进度事件格式

### 事件结构

```json
{
  "type": "workflow_progress",
  "data": {
    "workflow": "field-mapping",
    "node": "get_source_schema",
    "status": "started|completed|failed",
    "data": {
      "description": "正在获取源表结构",
      "source_db": "test_db",
      "source_table": "test_table"
    }
  }
}
```

### 状态说明

- **started**: 节点开始执行
- **completed**: 节点执行成功
- **failed**: 节点执行失败

## 前端集成指南

### 1. 监听workflow_progress事件

```typescript
// 在SSE事件处理中添加
eventSource.addEventListener('message', (event) => {
  const data = JSON.parse(event.data);
  
  if (data.type === 'workflow_progress') {
    const progress = data.data;
    
    // 更新进度UI
    updateWorkflowProgress({
      workflow: progress.workflow,
      node: progress.node,
      status: progress.status,
      description: progress.data.description
    });
  }
});
```

### 2. 进度UI示例

```tsx
interface WorkflowProgress {
  workflow: string;
  node: string;
  status: 'started' | 'completed' | 'failed';
  description: string;

  const nodes = [

    { id: 'run_mapping_script', label: '运行映射脚本' },
    { id: 'read_ddl', label: '读取DDL' },
    { id: 'validate', label: '验证结果' }
  ];
  
  return (
    <Card>
      <Title>工作流执行进度</Title>
      <Stepper active={getCurrentStep(progress.node)}>
        {nodes.map((node) => (
          <Stepper.Step
            key={node.id}
            label={node.label}
            loading={progress.node === node.id && progress.status === 'started'}
            color={getStepColor(node.id, progress)}
          />
        ))}
      </Stepper>
      <Text size="sm" c="dimmed">{progress.description}</Text>
    </Card>
  );
}
```

## 技术亮点

1. **非侵入式设计**: 通过回调机制,不影响工作流核心逻辑
2. **统一接口**: `emit_progress`函数提供统一的进度发送接口
3. **灵活扩展**: 新增工作流只需调用`emit_progress`即可
4. **实时推送**: 利用SSE流式传输,前端实时更新
5. **结构化数据**: JSON格式,易于解析和展示

## 使用示例

### 在新工作流中添加进度

```python
from app.workflows.base import emit_progress

def node_example(state):
    """示例节点"""
    # 节点开始
    emit_progress("my-workflow", "example_node", "started", {
        "description": "正在执行示例操作"
    })
    
    try:
        # 执行业务逻辑
        result = do_something()
        
        # 节点成功
        emit_progress("my-workflow", "example_node", "completed", {
            "description": "示例操作完成",
            "result_count": len(result)
        })
        
        return {"result": result}
        
    except Exception as e:
        # 节点失败
        emit_progress("my-workflow", "example_node", "failed", {
            "description": "示例操作失败",
            "error": str(e)
        })
        
        return {"errors": [str(e)]}
```
