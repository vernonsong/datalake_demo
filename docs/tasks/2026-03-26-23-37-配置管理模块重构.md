# 配置管理模块重构

**任务时间**: 2026-03-26 23:37

## 任务目标

检查并重构app后端的配置管理模块，确保除ENV参数外，所有其他配置都从YAML配置文件获取，而不是从环境变量获取。同时验证配置管理的4个核心功能。

## 需求分析

1. **配置来源限制**: 只允许ENV参数从环境变量获取，其他所有配置必须从YAML文件读取
2. **功能验证点**:
   - YAML配置读取
   - 多YAML配置合并
   - 不同环境YAML配置选装
   - 从应用配置中心获取配置

## 实施步骤

### 1. 问题识别

原有 `app/settings.py` 存在以下问题：
- 所有配置都从环境变量读取（APP_NAME, APP_VERSION, DEBUG, HOST, PORT等）
- 违反了"只有ENV从环境变量获取"的设计原则
- 存在循环依赖：settings依赖config，config又依赖settings

### 2. 架构重构

#### 2.1 Settings模块重构

**文件**: `app/settings.py`

**改动**:
- 移除所有配置子类（AppSettings, ServerSettings等）
- Settings类只保留`_env`属性从环境变量读取
- 所有其他配置通过`_load_config()`方法从YAML文件动态加载
- 使用property装饰器提供配置访问接口

**关键代码**:
```python
class Settings:
    def __init__(self):
        from dotenv import load_dotenv
        load_dotenv()
        
        self._env = os.getenv("ENV")
        if not self._env:
            raise ValueError("ENV environment variable is required")
        
        self._config: Optional[Dict[str, Any]] = None
    
    def _load_config(self) -> Dict[str, Any]:
        if self._config is None:
            from app.config import get_config
            from app.core.dependencies import get_token
            
            first_pass_config = get_config(env=self._env)
            
            config_service_url = first_pass_config.get("mock_service", {}).get("url")
            config_service_token = get_token() if config_service_url else None
            
            if config_service_url:
                self._config = get_config(
                    env=self._env,
                    config_service_url=config_service_url,
                    config_service_token=config_service_token
                )
            else:
                self._config = first_pass_config
        return self._config
```

#### 2.2 Config模块优化

**文件**: `app/config.py`

**改动**:
- `get_config()`函数增加参数：env, config_service_url, config_service_token
- 移除对settings的直接依赖，避免循环依赖
- 支持两阶段配置加载：先加载基础配置，再加载带配置中心的完整配置

#### 2.3 TokenProvider重构

**文件**: `app/core/dependencies.py`

**改动**:
- 延迟加载配置，避免初始化时的循环依赖
- 添加`_ensure_config()`方法按需加载配置
- 直接调用`get_config()`而不依赖settings

## 架构改进

### 配置加载流程

```
1. 应用启动
   ↓
2. Settings初始化，从环境变量读取ENV
   ↓
3. 首次访问配置属性时触发_load_config()
   ↓
4. 第一阶段：加载YAML配置（不含配置中心）
   ↓
5. 从YAML获取配置中心URL和获取token
   ↓
6. 第二阶段：重新加载配置（含配置中心）
   ↓
7. 替换${key}占位符为配置中心的值
   ↓
8. 缓存最终配置
```

### 依赖关系优化

**重构前**:
```
Settings → 环境变量（所有配置）
Config → Settings（循环依赖）
TokenProvider → Settings（循环依赖）
```

**重构后**:
```
Settings → 环境变量（仅ENV）
Settings → Config（延迟加载）
Config → 独立（无依赖）
TokenProvider → Config（延迟加载）
```

## 注意事项

1. **环境变量清理**: 可以从.env中移除除ENV外的其他配置项，但为了向后兼容暂时保留
2. **配置中心依赖**: 配置中心不可用时，占位符会保持原样，不会报错
3. **性能优化**: 配置只在首次访问时加载，后续访问使用缓存
4. **测试隔离**: 测试中使用mock避免依赖外部服务
