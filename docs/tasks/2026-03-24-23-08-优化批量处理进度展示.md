# 优化批量处理进度展示

**任务时间**: 2026-03-24 23:08

## 任务目标

优化批量处理功能的前端展示,让用户能够直观看到每个订单的处理过程和整体进度。

## 需求背景

当前批量处理功能存在以下问题:
1. 每个订单在独立子会话中执行,前端无法追踪进度
2. 用户只能看到主会话的消息,子会话的处理过程不可见
3. 缺少整体进度指示,用户不知道总共有多少订单,当前处理到第几个

## 设计方案

### 技术方案

采用**SSE流式推送 + 进度事件**方案:

1. **后端改造**:
   - `batch_tool.py`: 在处理过程中通过`print()`输出结构化的`[BATCH_PROGRESS]`标记
   - `chat_agent.py`: 在流式处理中捕获ToolMessage,解析`[BATCH_PROGRESS]`标记并转换为`batch_progress`事件
   - `chat.py`: 在SSE流中传递`batch_progress`事件

2. **前端改造**:
   - 新增`BatchProgress`组件: 显示批量处理进度卡片
   - `Chat.jsx`: 处理`batch_progress`事件并更新UI
   - 实时展示每个订单的状态(pending/processing/success/failed)

### 进度事件类型

```json

  "type": "batch_start",

{
```

## 实现内容

### 1. 后端实现

#### 1.1 batch_tool.py

**文件**: `app/agents/tools/batch_tool.py`

**修改内容**:
- 新增`BatchProgressCallback`类,用于推送进度事件
- 在`batch_process`函数中:
  - 处理开始时输出`batch_start`事件
  - 每个订单开始时输出`item_start`事件
  - 每个订单完成时输出`item_complete`事件
  - 每个订单失败时输出`item_error`事件
  - 批次完成时输出`batch_complete`事件

**关键代码**:
```python
class BatchProgressCallback:
    def on_item_start(self, index: int, total: int, item: Dict[str, Any]):
        event = {"type": "item_start", "index": index, "total": total, "item": item}
        print(f"\n[BATCH_PROGRESS] {json.dumps(event, ensure_ascii=False)}")
    
    def on_item_complete(self, index: int, total: int, item: Dict[str, Any], status: str, message: str = ""):
        event = {"type": "item_complete", "index": index, "total": total, "item": item, "status": status, "message": message}
        print(f"\n[BATCH_PROGRESS] {json.dumps(event, ensure_ascii=False)}")
```

#### 1.2 chat_agent.py

**文件**: `app/agents/chat_agent.py`

**修改内容**:
- 在`chat_stream`方法中,处理ToolMessage时解析`[BATCH_PROGRESS]`标记
- 将解析出的进度数据作为`batch_progress`事件yield出去

**关键代码**:
```python
if 'ToolMessage' in msg_type:
    if hasattr(msg, 'content') and msg.content:
        content_str = str(msg.content)
        import re
        progress_matches = re.findall(r'\[BATCH_PROGRESS\]\s*(\{.*?\})', content_str, re.DOTALL)
        for progress_json in progress_matches:
            try:
                progress_data = json.loads(progress_json)
                yield {'type': 'batch_progress', 'data': progress_data}
            except json.JSONDecodeError:
                pass
    continue
```

#### 1.3 chat.py

**文件**: `app/routers/chat.py`

**修改内容**:
- 在`stream_generator`中添加`batch_progress`事件的SSE推送

**关键代码**:

```python
elif event_type == 'batch_progress':
    progress_data = event.get('data', {})
```

### 2. 前端实现

#### 2.1 BatchProgress组件

**文件**: `frontend/src/components/Chat.jsx`

**新增内容**:
- `BatchProgress`组件: 展示批量处理进度
  - 显示总体进度(已处理/总数, 百分比)
  - 显示成功/失败统计
  - 列表展示每个订单的状态和详情
  - 支持折叠/展开

**关键特性**:
- 实时更新: 每收到一个进度事件就更新UI
- 状态图标: pending(⏸️), processing(⏳), success(✅), failed(✗)
- 进度条: 显示百分比和统计信息
- 错误提示: 失败订单显示错误信息

#### 2.2 Chat组件状态管理

**文件**: `frontend/src/components/Chat.jsx`

**修改内容**:
- 新增`batchProgressRef`状态管理批量处理进度
- 在事件循环中处理`batch_progress`事件:
  - `batch_start`: 初始化进度数据
  - `item_start`: 更新订单状态为processing
  - `item_complete`: 更新订单状态为success/completed
  - `item_error`: 更新订单状态为failed
- 将批量处理进度作为特殊消息类型插入消息列表

#### 2.3 Message组件

**文件**: `frontend/src/components/Chat.jsx`

**修改内容**:
- 添加对`batch_progress`类型消息的渲染
- 调用`BatchProgress`组件展示进度

## 技术亮点

1. **非侵入式设计**: 通过print输出和正则解析,不需要修改LangGraph核心流程
2. **实时推送**: 利用SSE流式传输,前端实时看到进度更新
3. **状态隔离**: 批量处理进度独立于思考过程,互不干扰
4. **用户体验**: 清晰的进度指示,让用户了解处理状态
