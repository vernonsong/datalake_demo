# 修复人工介入增删改接口

**任务时间**: 2026-03-26 00:00

## 问题描述

用户反馈：人工介入增删改类接口未生效。

## 问题分析

### 1. 原始配置问题

在 `app/core/dependencies.py` 中，原始配置如下：

```python
interrupt_on={
    "write_file": True,
    "platform_service": {
        "condition": _should_interrupt_platform_service
    },
}
```

### 2. 根本原因

通过分析 `langchain.agents.middleware.human_in_the_loop.HumanInTheLoopMiddleware` 源码发现：

```python
def __init__(self, interrupt_on: dict[str, bool | InterruptOnConfig], ...):
    resolved_configs: dict[str, InterruptOnConfig] = {}
    for tool_name, tool_config in interrupt_on.items():
        if isinstance(tool_config, bool):
            if tool_config is True:
                resolved_configs[tool_name] = InterruptOnConfig(...)
        elif tool_config.get("allowed_decisions"):  # 关键：只有包含allowed_decisions的配置才会被处理
            resolved_configs[tool_name] = tool_config
    self.interrupt_on = resolved_configs
```

**问题**：`HumanInTheLoopMiddleware` 不支持动态判断函数，只支持：
1. `bool` 类型（True/False）
2. 包含 `allowed_decisions` 字段的 `InterruptOnConfig` 字典

原始配置中的 `{"condition": _should_interrupt_platform_service}` 既不是 bool，也没有 `allowed_decisions` 字段，因此被**完全忽略**了。

## 解决方案

### 实现自定义 `DynamicHumanInTheLoopMiddleware`

创建一个继承自 `HumanInTheLoopMiddleware` 的自定义中间件，支持动态判断逻辑：

```python
class DynamicHumanInTheLoopMiddleware(HumanInTheLoopMiddleware):
    """支持动态判断逻辑的 HumanInTheLoopMiddleware"""
    
    def __init__(self, interrupt_on: dict, dynamic_conditions: dict):
        super().__init__(interrupt_on=interrupt_on)
        self.dynamic_conditions = dynamic_conditions

    def after_model(self, state, runtime):
        # 检查是否有需要动态中断的工具调用
        messages = state["messages"]
        if not messages:
            return None
            
        last_ai_msg = next((msg for msg in reversed(messages) if isinstance(msg, AIMessage)), None)
        if not last_ai_msg or not last_ai_msg.tool_calls:
            return None

        # 动态更新 interrupt_on 配置
        current_interrupt_on = self.interrupt_on.copy()
        
        for tool_call in last_ai_msg.tool_calls:
            tool_name = tool_call["name"]
            if tool_name in self.dynamic_conditions:
                condition_func = self.dynamic_conditions[tool_name]
                try:
                    if condition_func(tool_call):
                        # 如果条件满足，添加中断配置
                        self.interrupt_on[tool_name] = InterruptOnConfig(
                            allowed_decisions=["approve", "reject"]
                        )
                except Exception as e:
                    logger.error(f"动态中断判断失败: {e}")
        
        try:
            # 调用父类逻辑
            return super().after_model(state, runtime)
        finally:
            # 恢复配置
            self.interrupt_on = current_interrupt_on
```

### 更新 `get_deep_agent` 配置

```python
# 静态中断配置
static_interrupt_on = {
    "write_file": True,
    "read_file": False,
    "edit_file": False,
    "execute_command": False,
    "create_directory": False,
}

# 动态中断条件
dynamic_conditions = {
    "platform_service": _should_interrupt_platform_service
}

# 创建动态中间件
dynamic_middleware = DynamicHumanInTheLoopMiddleware(
    interrupt_on=static_interrupt_on,
    dynamic_conditions=dynamic_conditions
)

return create_deep_agent(
    # ...
    interrupt_on=None,  # 不使用内置的 HumanInTheLoopMiddleware
    middleware=[dynamic_middleware],  # 使用自定义的动态中间件
    # ...
)
```

## 工作原理

1. **初始化阶段**：
   - `DynamicHumanInTheLoopMiddleware` 初始化时，只包含静态配置（如 `write_file: True`）
   - `platform_service` 不在 `interrupt_on` 中，因此默认不会中断

2. **运行时判断**：
   - 当 LLM 生成工具调用时，`after_model` 被触发
   - 遍历所有工具调用，检查是否在 `dynamic_conditions` 中
   - 如果是 `platform_service`，调用 `_should_interrupt_platform_service(tool_call)` 判断
   - 如果返回 `True`，临时将 `platform_service` 添加到 `self.interrupt_on`

3. **中断触发**：
   - 调用父类的 `after_model`，它会检查 `self.interrupt_on`
   - 发现 `platform_service` 在配置中，触发 `interrupt()`
   - LangGraph 暂停执行，等待用户确认

4. **恢复配置**：
   - 在 `finally` 块中恢复原始配置，确保下次调用时重新判断

## 判断逻辑

`_should_interrupt_platform_service` 函数通过读取接口文档中的 `REQUIRES_CONFIRMATION` 标记来判断：

```python
def _should_interrupt_platform_service(tool_call: dict) -> bool:
    args = tool_call.get("args", {})
    doc_path = args.get("doc_path", "")
    
    if not doc_path:
        return False
    
    try:
        resolved_path = _resolve_doc_path(doc_path)
        content = resolved_path.read_text(encoding="utf-8")
        match = re.search(r'REQUIRES_CONFIRMATION:\s*(true|false)', content, re.IGNORECASE)
        
        if match:
            return match.group(1).lower() == 'true'
        else:
            return False  # 向后兼容：未标记的默认不需要确认
    except Exception as e:
        logger.warning(f"读取接口文档失败: {doc_path}, 错误: {e}")
        return True  # 安全优先：出错时默认需要确认
```
