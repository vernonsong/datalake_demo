# 如何触发工作流流式进度

## 前端触发方式

### 1. 通过聊天界面

在前端聊天界面中,输入以下消息即可触发工作流执行并看到流式进度:

#### 测试工作流
```
执行test-streaming工作流,参数: {"data": {"text": "测试数据"}}
```

或者更自然的表达:
```
帮我运行test-streaming工作流,输入数据是"测试数据"
```

#### 字段映射工作流
```
执行field-mapping工作流,参数: {
  "order_id": "ORDER001",
  "source_db": "source_db",
  "source_table": "orders",
  "target_db": "clickhouse",
  "target_table": "dw_orders"
}
```

### 2. 前端显示效果

当工作流执行时,前端会实时显示:

```
🔄 工作流: test-streaming
[4/4] 100%

▼ 展开查看详情:
  ✓ data_preparation (步骤 1)
  ✓ data_validation (步骤 2)
  ✓ data_processing (步骤 3)
  ✓ result_formatting (步骤 4)
```

**特点**:
- ✅ 实时更新进度
- ✅ 显示当前执行的节点
- ✅ 显示完成百分比
- ✅ 可折叠/展开查看详细步骤
- ✅ 紫色主题,与批量处理进度区分

### 3. 前端组件说明

**WorkflowProgress 组件**位于 `frontend/src/components/Chat.jsx`:

```jsx
<WorkflowProgress data={{
  workflow_name: "test-streaming",
  nodes: [
    { node_name: "data_preparation", status: "completed", step: 1 },
    { node_name: "data_validation", status: "completed", step: 2 },
    { node_name: "data_processing", status: "processing", step: 3 },
    { node_name: "result_formatting", status: "pending", step: 4 }
  ]
}} />
```

**状态说明**:
- `pending` - 等待执行(灰色圆圈)
- `processing` - 正在执行(紫色旋转圆圈)
- `completed` - 已完成(绿色对勾)

## 后端API

### 直接调用API

```bash
curl -N -X POST http://localhost:8000/chat/ \
  -F "message=执行test-streaming工作流,参数: {\"data\": {\"text\": \"测试数据\"}}" \
  -F "stream=true"
```

### SSE事件格式

工作流进度事件格式:
```json
data: {
  "type": "workflow_progress",
  "workflow_name": "test-streaming",
  "node_name": "data_preparation",
  "status": "completed",
  "step": 1
}
```

## 可用的工作流

### 1. test-streaming
**位置**: `skills/scenario-skill/test-streaming/`

**用途**: 测试流式进度输出

**参数**:
```json
{
  "data": {
    "text": "任意文本"
  }
}
```

**节点**:
1. data_preparation - 数据准备(2秒)
2. data_validation - 数据验证(2秒)
3. data_processing - 数据处理(2秒)
4. result_formatting - 结果格式化(2秒)

### 2. field-mapping
**位置**: `skills/scenario-skill/field-mapping/`

**用途**: 字段映射工作流

**参数**:
```json
{
  "order_id": "ORDER001",
  "source_db": "source_db",
  "source_table": "orders",
  "target_db": "clickhouse",
  "target_table": "dw_orders"
}
```

## 技术实现

### 流式输出原理

1. **异步工具** - `execute_workflow` 使用 `async def`
2. **astream()** - 使用 `workflow.astream()` 异步迭代
3. **回调机制** - 通过 `_emit_progress()` 实时发送进度
4. **SSE推送** - 通过Server-Sent Events实时推送到前端

### 关键文件

**后端**:
- `app/agents/tools/workflow_tool.py` - 异步工作流工具
- `app/agents/chat_agent.py` - 进度回调处理
- `app/routers/chat.py` - SSE事件发送

**前端**:
- `frontend/src/components/Chat.jsx` - WorkflowProgress组件

## 故障排查

### 前端没有显示进度

1. 检查浏览器控制台是否有错误
2. 检查Network标签,查看SSE事件是否收到
3. 确认后端日志中有 "📤 进度事件已发送" 日志

### 后端没有发送进度

1. 检查工作流是否正确加载: `curl http://localhost:8000/api/v1/workflows/`
2. 检查后端日志中是否有 "🔍 _emit_progress called"
3. 确认回调已设置: 查找 "✅ 工作流进度回调已设置" 日志

### 工作流执行失败

1. 检查工作流配置文件是否正确
2. 检查工作流脚本是否有语法错误
3. 查看后端日志中的错误堆栈
