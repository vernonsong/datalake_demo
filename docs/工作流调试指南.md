# 工作流调试指南

## 问题：为什么断点打在 execute_workflow 进不去？

### 原因分析

`execute_workflow` 函数被 LangChain 的 `@tool` 装饰器包装，实际调用时经过了多层包装：

```python
# app/workflows/tool.py
@tool
async def execute_workflow(workflow_name: str, params: Dict[str, Any]) -> Dict[str, Any]:
    # 你的代码
    ...
```

**调用链路**：
```
智能体 
  ↓
tool.ainvoke() (LangChain 包装层)
  ↓
装饰器内部逻辑
  ↓
你的 execute_workflow 函数  ← 断点应该打在这里的函数体内
```

### 解决方案

## 方法1：在函数体内打断点（推荐）

**不要在函数定义行打断点**，而是在函数体的第一行或关键逻辑行打断点：

```python
@tool
async def execute_workflow(workflow_name: str, params: Dict[str, Any]) -> Dict[str, Any]:
    """..."""
    try:
        workflow_json = load_workflow_definition(workflow_name)  # ← 在这里打断点
        
        graph = compile_workflow_to_langgraph(workflow_json)  # ← 或这里
```

- ✅ [tool.py:35](file:///Users/vernonsong/Documents/项目/datalake_demo/app/workflows/tool.py#L35) - `workflow_json = load_workflow_definition(workflow_name)`
- ✅ [tool.py:37](file:///Users/vernonsong/Documents/项目/datalake_demo/app/workflows/tool.py#L37) - `graph = compile_workflow_to_langgraph(workflow_json)`

## 方法4：使用 Python 内置调试器

在代码中插入 `breakpoint()`：

```python
@tool
async def execute_workflow(workflow_name: str, params: Dict[str, Any]) -> Dict[str, Any]:
    """..."""
    breakpoint()  # ← 程序会在这里暂停，进入 pdb 调试器
    
    try:
        workflow_json = load_workflow_definition(workflow_name)
        # ...
```

然后运行程序，会自动进入 pdb 交互式调试：


## 调试技巧

### 1. 禁用 justMyCode

在 VSCode 调试配置中设置 `"justMyCode": false`，这样可以进入第三方库代码（如 LangChain、LangGraph）进行调试。

### 2. 使用条件断点

在 VSCode 中右键断点 → "编辑断点" → 添加条件：

```python
workflow_name == "test_basic"
```

这样只有当 `workflow_name` 为 `"test_basic"` 时才会暂停。

### 3. 使用日志断点

在 VSCode 中右键断点 → "编辑断点" → 选择 "日志消息"：

```
workflow_name={workflow_name}, params={params}
```

这样不会暂停程序，只会打印日志。

### 4. 查看调用栈

当断点命中时，在 VSCode 的 "调用堆栈" 面板中可以看到完整的调用链路：

```
execute_workflow (tool.py:35)
  ← ainvoke (langchain/tools/base.py:xxx)
    ← _call_tool (deepagents/agent.py:xxx)
      ← invoke (langgraph/graph.py:xxx)
        ← chat_stream (chat_agent.py:xxx)
```

## 故障排查

### 问题1：断点显示灰色（未绑定）

**原因**：源代码路径不匹配

**解决**：
- 确保 `PYTHONPATH` 包含项目根目录
- 检查 `.vscode/launch.json` 中的 `env.PYTHONPATH` 配置

### 问题2：断点不命中

**原因**：代码没有被执行到

**解决**：
- 检查工作流是否真的被调用（查看日志）
- 尝试在更早的位置打断点（如 `chat_agent.py`）
- 使用 `breakpoint()` 强制暂停

### 问题3：调试器启动失败

**原因**：端口被占用或环境问题

**解决**：
```bash
# 检查端口占用
lsof -i :8000

# 杀死占用进程
kill -9 <PID>

# 重新启动调试
```

3. ✅ 结合日志和断点一起使用
